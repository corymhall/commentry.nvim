*commentry.txt*  Local diff review workflows for Neovim

INTRODUCTION                                     *commentry*

commentry.nvim provides local diff review workflows inside Neovim with
draft persistence, typed/range comments, reviewed-file tracking, and an
optional Codex handoff path.
Minimum Neovim version: 0.10+.

INSTALL                                           *commentry.install*

Required dependencies:
- Neovim 0.10+
- `diffview.nvim` for the diff UI

Optional integrations:                           *commentry.dependencies*
- `snacks.nvim` for `:Commentry list-comments`
- Sidekick runtime for `:Commentry send-to-codex`

SETUP                                             *commentry.setup*

Use the Lua API to configure the plugin:
>
  require("commentry").setup({
    log = {
      level = "warn",  -- error|warn|info|debug
      sink = "notify", -- notify|echo|file
      file = nil,      -- path when sink == "file"
    },
    diagnostics = {
      open_style = "split", -- split|vsplit|float
    },
    diffview = {
      auto_attach = true,
      comment_cards = {
        max_width = 88,
        max_body_lines = 8,
        show_markers = true,
      },
      comment_ranges = {
        enabled = true,
        line_highlight = true,
      },
    },
  })
<

KEYMAP CONFIG                                     *commentry.keymaps*

All keymaps are configured under `setup({ keymaps = { ... } })` and attach
buffer-locally only in Commentry diffview buffers.

Action matrix:

  Action                  Default   Mode     Empty disable   Command fallback
  add_comment             mc        Normal   No              None
  add_range_comment       mc        Visual   No              :Commentry add-range-comment
  edit_comment            me        Normal   No              None
  delete_comment          md        Normal   No              None
  set_comment_type        mt        Normal   No              :Commentry set-comment-type
  toggle_file_reviewed    mr        Normal   Yes (`""`)      :Commentry toggle-file-reviewed
  next_unreviewed_file    ]r        Normal   Yes (`""`)      :Commentry next-unreviewed
  send_to_codex           ms        Normal   No              :Commentry send-to-codex
  list_comments           ml        Normal   No              :Commentry list-comments

Behavior notes:
- Empty-string disable is intentionally supported only for
  `toggle_file_reviewed` and `next_unreviewed_file`.
- For remap-only actions (`add_comment`, `add_range_comment`, `edit_comment`,
  `delete_comment`, `set_comment_type`, `send_to_codex`, `list_comments`),
  `""` is invalid.
  Setup warns and
  the effective mapping remains active.
- `add_range_comment` normally uses its configured/default value from setup
  normalization. The fallback chain to resolved `add_comment` (then `mc`) is
  a defensive runtime path when `Config.keymaps` is missing or bypasses
  normalization.

Example (partial remap + selective disable):
>
  require("commentry").setup({
    keymaps = {
      add_comment = "gc",
      add_range_comment = "gc",
      edit_comment = "ge",
      delete_comment = "gd",
      set_comment_type = "gt",
      send_to_codex = "gs",
      list_comments = "gl",
      toggle_file_reviewed = "",
      next_unreviewed_file = "]u",
    },
  })
<

Example (change one mapping, keep other defaults):
>
  require("commentry").setup({
    keymaps = {
      next_unreviewed_file = "]n",
    },
  })
<

COMMANDS                                          *commentry.commands*

:Commentry
  Base command entry point for commentry. Subcommands will be added as
  the plugin evolves.

:Commentry open
  Open diffview for local changes.

:Commentry add-range-comment
  Create a range comment from the current visual selection.

:Commentry list-comments
  List jumpable active draft comments for the current diff file/side.
  Requires `snacks.nvim`.

:Commentry set-comment-type
  Set default or per-comment type.

:Commentry toggle-file-reviewed
  Toggle reviewed status for the current diff file.

:Commentry next-unreviewed
  Jump to the next unreviewed file in the active diffview file list.

:Commentry export [stdout|register|register:{name}]
  Export active draft comments as deterministic markdown.
  `stdout` (default): print markdown.
  `register`: write markdown to unnamed register.
  `register:{name}`: write markdown to register {name} (example: `register:a`).

:Commentry debug-store
  Print current review context + on-disk store details:
  - context id/mode/revisions
  - project root
  - store path
  - whether the store file exists
  Useful when troubleshooting persistence across sessions/contexts.

:Commentry diagnostics
  Open a scratch diagnostics buffer containing:
  - Neovim version
  - active logger config
  - codex enabled state
  - resolved store/context metadata
  - diffview attach/buffer state

:Commentry send-to-codex
  Send current review payload to Codex.
  Requires:
  - codex integration enabled in setup (`codex.enabled = true`)
  - available Sidekick runtime/session target
  - active attached review context (for example via `:Commentry open` or diffview auto-attach)
  Adapter behavior:
  - v1 adapter selection is global/implicit from config
  - when exactly one existing Codex Sidekick session exists, send auto-attaches it
  - when multiple existing sessions exist but exactly one matches current cwd, send auto-selects it
  - when multiple existing sessions exist, send prompts for target selection
  - when no existing sessions exist, send fails with actionable guidance
  Semantics:
  - send-and-forget: one dispatch attempt, immediate success/failure feedback
  - payload format is compact text (`COMMENTRY_REVIEW_V1`), optimized for
    human readability and low token overhead
  - no persisted send history/receipts/retry queue in v1

V1 BOUNDARIES                                     *commentry.v1-boundaries*

- No persisted outbound send log/history
- No background retry queue or delivery reconciliation
- No receipt tracking inside commentry state
- No multi-adapter orchestration in v1

PERSISTENCE NOTES                                 *commentry.persistence*

- Store path is context-scoped:
  `~/.commentry/repos/<repo>/contexts/<context-id>/commentry.json`
- The store file is created lazily on first successful write in that context.
- Review context is branch-scoped (`<root>::review::branch::<branch-name>`) across
  Diffview range lenses (`:DiffviewOpen`, `:DiffviewOpen main`, `:DiffviewOpen HEAD~3`, etc)
  for that branch.
- Comments remain in the same context until anchor reconciliation marks them
  stale/outdated after code changes.
- Comment add/edit/range opens a floating multiline editor:
  - `Enter` inserts newline
  - `Ctrl-s` saves
  - `q` or `Esc` in normal mode cancels
  - `Tab` cycles comment type
- Comment bodies render as persistent boxed virtual cards on commented lines;
  they stay visible when moving the cursor.
- Use `:Commentry debug-store` to verify the active context/path when comments seem missing.

TROUBLESHOOTING                               *commentry.troubleshooting*

- `:Commentry list-comments` is unavailable:
  install `snacks.nvim`, then rerun `:checkhealth commentry`.
- `:Commentry send-to-codex` has no target:
  ensure a Sidekick session is attached, then retry from an attached review
  buffer/context.
