*commentry.txt*  AI review scaffolding for Neovim

INTRODUCTION                                     *commentry*

commentry.nvim is a Neovim plugin scaffold intended for building
PR-style review workflows for AI-generated code. This doc is minimal
and will expand as features are added.

SETUP                                             *commentry.setup*

Use the Lua API to configure the plugin:
>
  require("commentry").setup({
    diffview = {
      comment_cards = {
        max_width = 88,
        max_body_lines = 8,
        show_markers = true,
      },
    },
  })
<

COMMANDS                                          *commentry.commands*

:Commentry
  Base command entry point for commentry. Subcommands will be added as
  the plugin evolves.

:Commentry open
  Open diffview for local changes.

:Commentry list-comments
  List jumpable active draft comments for the current diff file/side.

:Commentry set-comment-type
  Set default or per-comment type.

:Commentry toggle-file-reviewed
  Toggle reviewed status for the current diff file.

:Commentry next-unreviewed
  Jump to the next unreviewed file in the active diffview file list.

:Commentry export [stdout|register|register:{name}]
  Export active draft comments as deterministic markdown.
  `stdout` (default): print markdown.
  `register`: write markdown to unnamed register.
  `register:{name}`: write markdown to register {name} (example: `register:a`).

:Commentry debug-store
  Print current review context + on-disk store details:
  - context id/mode/revisions
  - project root
  - store path
  - whether the store file exists
  Useful when troubleshooting persistence across sessions/contexts.

:Commentry send-to-codex
  Send current review payload to Codex.
  Requires:
  - codex integration enabled in setup (`codex.enabled = true`)
  - active attached review context (for example via `:Commentry open` or diffview auto-attach)
  Adapter behavior:
  - v1 adapter selection is global/implicit from config and attached runtime target
  Semantics:
  - send-and-forget: one dispatch attempt, immediate success/failure feedback
  - payload format is compact text (`COMMENTRY_REVIEW_V1`), optimized for
    human readability and low token overhead
  - no persisted send history/receipts/retry queue in v1

V1 BOUNDARIES                                     *commentry.v1-boundaries*

- No persisted outbound send log/history
- No background retry queue or delivery reconciliation
- No receipt tracking inside commentry state
- No multi-adapter orchestration in v1

PERSISTENCE NOTES                                 *commentry.persistence*

- Store path is context-scoped:
  `~/.commentry/repos/<repo>/contexts/<context-id>/commentry.json`
- The store file is created lazily on first successful write in that context.
- Review context is repository-scoped (`<root>::review`) across Diffview range
  lenses (`:DiffviewOpen`, `:DiffviewOpen main`, `:DiffviewOpen HEAD~3`, etc).
- Comments remain in the same context until anchor reconciliation marks them
  stale/outdated after code changes.
- Comment add/edit/range opens a floating multiline editor:
  - `Enter` inserts newline
  - `Ctrl-s` saves
  - `q` or `Esc` in normal mode cancels
  - `Tab` cycles comment type
- Comment bodies render as persistent boxed virtual cards on commented lines;
  they stay visible when moving the cursor.
- Use `:Commentry debug-store` to verify the active context/path when comments seem missing.
